    欢迎使用 Git Save/Load

    🔐
    
    
    先设个身份再提交
    
    
    
      Git 要知道“谁”提交了这次修改。填下面两项，只保存在当前仓库里，不会上传到任何地方。

    📂 变更文件 — 文件状态说明
    
  
    点击文件名可查看具体变更内容（diff）
    
  
    
    ??未跟踪（新文件）
    M已修改
    A已暂存的新文件
    D已删除
    R重命名
    U合并冲突
  
    

    



    
  
    📜 提交记录 — 回滚模式说明
    
  
    点击 commit hash 或回滚按钮可选择回滚模式
    
  
    
    soft
    仅移动 HEAD 指针，代码和暂存区不变
    mixed
    移动 HEAD + 清空暂存区，代码保留
    hard
    ⚠ 彻底还原，未提交的修改全部丢失
  
    

    



    
  
    
    
    ↩️ 回滚到指定版本
    
    
    
    
    
    
      
    回滚到什么程度？
    
      
    
        
        
    
          
    🟡
    
          
    
            
    soft — 轻撤回（推荐）
    
            
    只撤回 commit 记录，代码和暂存区不变。
    适合：提交后发现消息写错了，或想合并成一次提交。
    
          
    
        
    
        
        
    
          
    🔵
    
          
    
            
    mixed — 清暂存（最常用）
    
            
    撤回 commit 并清空暂存区，代码保留在工作区。
    适合：提交后发现漏了文件，清暂存后重新 add 再提交。默认选项。
    
          
    
        
    
        
        
    
          
    🔴
    
          
    
            
    hard — 硬重置 ⚠️ 慎用
    
            
    彻底回到目标版本，未提交的修改全部丢失。
    除非你确定不要这些修改了，否则不要选这个！
